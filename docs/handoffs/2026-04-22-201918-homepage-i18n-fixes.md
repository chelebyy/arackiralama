### Key Patterns Discovered

- **next-intl `Link` strict typing**: When `pathnames` is configured in `routing.ts`, `Link` only accepts known static paths. Dynamic routes MUST be defined in `pathnames` (e.g. `/vehicles/[id]`) and passed as objects: `href={{ pathname: "/vehicles/[id]", params: { id } }}`.
- **Do NOT mix `NextLink` and i18n `Link` casually**: VehicleCard originally used raw `next/link` which bypassed locale prefixing, causing links to go to `/vehicles/1` instead of `/tr/araclar/1`.
- **Native `<details>` for accordion**: Used instead of importing a heavy UI library component. Clean, accessible, zero-JS-failure.
- **Turkish text + `uppercase` = visual bug**: Turkish characters (İ, Ş, Ç, Ğ) render poorly with `text-transform: uppercase`. Removed from footer headings.

## Context for Resuming Agent

### Important Context

- **Always use i18n `Link` from `@/i18n/routing`** for internal navigation. If linking to a dynamic route, define it in `routing.ts` `pathnames` and pass `{ pathname, params }`.
- **The Turkish translation file (`tr.json`) is now the source of truth** for homepage content. Any future homepage text changes must go there first, then referenced via `useTranslations("home")`.
- **Build must pass before claiming completion**: `corepack pnpm -C frontend build` is the verification gate. next-intl type errors are strict.
- **The public site must NOT use shadcn/ui components** per project AGENTS.md design rules. Use Tailwind + native HTML + lucide-react.

### Assumptions Made

- The Turkish translations added are acceptable copy (not professionally proofread).
- FAQ content in `tr.json` is accurate to business requirements.
- Vehicle detail pages (`/tr/araclar/[id]`) are implemented elsewhere and functional.

### Potential Gotchas

- **next-intl `Link` type strictness**: Passing a plain string like `/vehicles/1` to i18n `Link` will cause a TypeScript build failure if `pathnames` is configured. Always use the object form for dynamic routes.
- **LanguageSwitcher on dynamic pages**: The i18n `Link` with `locale` prop breaks on dynamic routes. Current workaround uses raw `NextLink` with manual path prefix stripping. If changing route structure, this logic must be updated.
- **Turkish uppercase CSS**: Never apply `uppercase` or `uppercase tracking-wider` to Turkish headings — İ, Ş, Ç, Ğ characters render incorrectly.
- **`t.raw()` usage**: When using `t.raw("faq.questions")`, ensure the JSON value is an array. If a translation is missing, it returns the key string which will break `.map()`.
